## Where This Fits In

If Pre-Security was the foundation — getting comfortable with the basics of how computers, networks, and operating systems work — Cyber Security 101 is where things start to get real. This path takes everything from Pre-Security and builds on it with more depth, more tools, and more hands-on scenarios that actually feel like security work.

Coming into this path most of the jargon already made sense to me. Terms like subnetting, ports, protocols, file permissions, SQL — I'd seen all of it before between the Google Cybersecurity Certificate and the Pre-Security path. What's different here is the application. It's less about understanding what something is and more about actually using it, probing it, and thinking about it from both an attacker and defender perspective.

This is where I started feeling like I was genuinely doing security work rather than just studying it.

## How These Files Are Written

Every module and room in this folder follows the same format I've used across all my certifications — no syllabus summaries, no copying what the course says. Each file covers:

- **What I actually did** — the specific commands I ran, the tasks I completed, the labs I worked through
- **What the output showed** — real terminal output, scan results, flags captured, documented with screenshots
- **What I learned** — the concepts that clicked, the things that took longer, the connections I made to other things I'd studied
- **My take** — an honest personal reflection on the room, what was interesting, what was confusing, and how it fits into the bigger picture

Screenshots are included in each module's `screenshots/` folder and referenced inline in the write-ups. These aren't staged — they're captures from the actual lab sessions as I worked through them.
